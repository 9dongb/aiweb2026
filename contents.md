# AI 회의 후속 업무 자동화 서비스

회의 음성 및 텍스트를 분석하여 핵심 요약, 결정사항, 담당자별 액션 아이템을 추출하고 실제 업무 실행(Notion 생성, 메일 전송, 캘린더 등록 등)으로 연결해주는 업무 보조 웹 서비스

---

## 한 줄 요약

사용자가 회의록 텍스트를 입력하면, AI가 요약과 함께 담당자별 액션 아이템을 추출하고, Notion 회의록 생성, 후속 공유 메일 발송, Google 캘린더 일정 등록까지 한 번에 처리해주는 서비스

---

## 문제 (목표)

회의 후에는 결정사항, 담당자별 할 일, 마감일이 명확히 정리되지 않아 후속 업무가 누락되거나 지연되는 일이 자주 발생합니다. 또한 회의록을 사람이 직접 정리하고, 팀원에게 공유하고, 일정을 등록하는 과정에 반복적인 시간이 많이 소요됩니다.

MeetingFlow AI의 목표는 단순한 회의록 요약이 아니라, 회의 내용을 실행 가능한 업무 데이터로 바꾸고 외부 업무 도구와 연결하는 자동화 파이프라인을 제공하는 것입니다.

| 구분 | 기존 방식의 문제 | MeetingFlow AI의 해결 방향 |
| --- | --- | --- |
| 회의록 정리 | 사람이 회의 내용을 다시 읽고 요약해야 함 | AI가 요약, 주요 논의, 결정사항을 자동 추출 |
| 후속 업무 관리 | 담당자와 마감일이 불명확해 누락 가능 | 액션 아이템을 담당자, 마감일, 우선순위, 상태와 함께 저장 |
| 공유 | 회의 후 공유 메일을 매번 새로 작성 | 분석 결과 기반 후속 이메일 초안 생성 |
| 일정 관리 | 캘린더 등록을 사람이 직접 처리 | 액션 아이템 마감일을 Google Calendar 일정으로 동기화 |
| 기록 관리 | 회의록과 업무 도구가 분리됨 | Notion 회의록 초안 생성으로 기록 연결 |
---

## 사용자

팀 프로젝트, 동아리, 수업 과제, 직장인처럼 회의 후 실행 항목을 정리해야 하는 사람과 팀을 주요 사용자로 봅니다.

---

## 핵심 기능

1. **회의 내용 구조화 및 요약**
   - 회의록 텍스트를 직접 입력하거나 `.txt`/`.docx` 파일을 업로드합니다.
   - AI가 회의 제목, 회의 날짜, 참석자, 요약, 주요 논의 내용, 결정사항, 액션 아이템, 미해결 이슈를 추출합니다.

2. **액션 아이템 관리**
   - 추출된 액션 아이템을 담당자, 마감일, 우선순위, 상태와 함께 저장합니다.
   - 사용자는 분석 결과를 검토하고 필요한 내용을 직접 수정할 수 있습니다.
   - 액션 아이템은 목록 또는 칸반 형태로 확인할 수 있습니다.

3. **후속 업무 실행**
   - 분석 결과를 바탕으로 후속 이메일 초안을 생성합니다.
   - 액션 아이템 마감일을 Google Calendar 일정으로 동기화합니다.
   - 회의 요약과 후속 업무를 Notion 회의록 초안으로 생성합니다.

4. **팀 기반 회의 관리**
   - 이메일/비밀번호 로그인과 Google OAuth 로그인을 지원합니다.
   - 사용자별 기본 팀을 만들고, 팀 단위로 회의와 액션 아이템을 공유합니다.

---

## 입력 → AI 처리 → 출력

사용자 회의록 텍스트 또는 `.txt`/`.docx` 파일 → FastAPI 백엔드가 회의 데이터를 저장하고, LLM API(OpenAI GPT-4o-mini)에 분석 프롬프트 호출 → 요약, 참석자, 주요 논의 내용, 결정사항, 액션 아이템, 후속 확인 사항 검증 →  대시보드에 분석 결과와 액션 아이템 보드 출력 + Google Calendar 일정 동기화, Notion 페이지 초안 생성, Email 초안 생성

| 입력 | AI 처리 | 출력 |
| --- | --- | --- |
| 사용자 회의록 텍스트 또는 `.txt`/`.docx` 파일 + 회의 제목/날짜/참석자 정보 | GPT API에 회의 분석 프롬프트 호출 → 회의록 분석 가능 여부 판단 → 요약, 주요 논의, 결정사항, 액션 아이템, 미해결 이슈를 JSON 스키마로 추출 | 회의 요약 + 결정사항 + 담당자별 액션 아이템 + 미해결 이슈 + 원문 근거 |
| 확정된 회의 분석 결과 + 참석자 이메일 | GPT API에 후속 이메일 작성 프롬프트 호출 → 분석 결과를 업무 공유 메일 형식으로 재구성 | 후속 이메일 초안 + 제목 + 수신자 후보 |

| 상황 | 사용 흐름 | 결과 |
| --- | --- | --- |

| 일정 관리 | 분석 결과 확인 → Google Calendar 동기화 | 담당 업무와 마감일이 캘린더에 등록 |
| 팀 공유 | 분석 결과 확인 → 후속 이메일 초안 생성 + 메일 앱으로 전송 | 팀원에게 동일한 회의 결과 공유 |
| 기록 보관 | 분석 결과 확인 → Notion 초안 생성 | 회의록과 결정사항을 Notion에 정리 |

## 기술 스택

| 영역 | 사용 기술 | 역할 |
| --- | --- | --- |
| 프론트엔드 | Next.js 15, React 19, TypeScript, Tailwind CSS | 회의 입력, 분석 결과 검토, 액션 보드 UI |
| 백엔드 | FastAPI, SQLAlchemy, SQLite | 인증, 회의 데이터 저장, 분석 결과 저장, 외부 연동 API |
| AI 모델/API | OpenAI Chat Completions API, `gpt-4o-mini` | 회의록 분석과 후속 이메일 초안 생성 |
| 인증 | 이메일/비밀번호, Google OAuth | 사용자 로그인과 팀 기반 접근 제어 |
| 외부 연동 | Google Calendar API, Notion OAuth/API, `mailto` | 일정 동기화, 회의록 초안 생성, 이메일 초안 작성 |

## 데이터 출처

사용자가 직접 입력하거나 업로드한 회의록 텍스트와 프로젝트에서 직접 작성한 데모 회의록을 사용합니다. 업로드 파일은 .txt와 .docx 회의록 문서이며, 저작권이 있거나 개인정보 동의가 없는 외부 회의록은 사용하지 않습니다.
